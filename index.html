Minesweeper

For this project, I made a Minesweeper game. Instead of using numbers to identify where the bombs are placed, I used colors.

  • Green signifies that there is no bomb in the eight squares surrounding the block
  • Yellow indicates that there is a bomb in one of those blocks
  • Red indicates that you have hit a bomb and lost the game

Concepts

For loops: to set the bombs up in the grid, to mark certain squares as "visited" so they are not rechecked
If-then statements: to check if the square had been visited, to check if there were any bombs around the square, to randomly assign bombs
Constants: to provide a way to check if a square had been checked or not to avoid Stack Overflow
Recursion: to call the checking method on each subsequently called square

Challenges

I kept running into a Stack Overflow error because the program would go back and forth between two squares or because it would circle back to the same square repeatedly. To prevent this, I marked squares as "visited" and added a while loop around the program to check if it had been visited before the loop executed.

In Action

Goals Achieved

The program runs and is a playable game. When a square is clicked, it changes its color based on whether or not it has been declared a "bomb." After being clicked, they other squares around it also change color and it continues until the squares turn yellow to signify a bomb nearby.

Future Work
